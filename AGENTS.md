# AutoLooper 作業ルール

## リリース運用

- GitHub Releases を作成・更新するときは、Release本文の先頭にユーザーがすぐ分かる直接ダウンロードリンクを置く。
- ダウンロードリンクは `Windows版アプリ本体` のように用途が分かる日本語ラベルにする。
- exe はGit履歴へ直接コミットせず、GitHub Releases のAssetとしてアップロードする。
- Release本文では、ダウンロードリンク、起動方法、チェックサム、主な変更点、検証結果の順に記載する。
- Asset欄だけに依存せず、本文内にも必ずexeへの直接リンクを書く。
